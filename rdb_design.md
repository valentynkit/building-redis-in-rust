Currently when slave connects to master, master add it to it clients array. However in such cases it will be mixed with other real clients that send commands, so on replication operation when master sends commands to all it slaves, it will have to find each time the slaves from array of clients. Ideally master should have a seperate array of slaves (clients) which will act as one way communication, it will not read from it only stream, still keeping the initial client in clients, for cases when slave sends REPLCONF ACK or something like that.
With this configuration we will have seperate slaves array of clients for one way stream from master to slave, but still have initial client in clients for pereodic two way communication.
